크로스체인 브릿지 (Cross-Chain Bridges)

서로 다른 블록체인 간에 자산·메시지·상태를 이전하기 위한 프로토콜. 각 체인은 독립된 합의·실행 환경을 가지므로 네이티브한 상호운용성이 존재하지 않으며, 브릿지는 그 격차를 메우는 중개 신뢰 계층이다. 2022~2023년 사이 단일 카테고리로는 가장 큰 해킹 피해(누적 25억 달러+)가 발생한 영역이기도 하다.

📚 참고: Vitalik: Multi-chain ≠ Cross-chain · Ethereum.org: Bridges · IBC Protocol · LayerZero V2 · Chainlink CCIP · rekt.news: 해킹 리더보드

3. 메시지 전달 추상화 (Generalized Messaging)

자산 이동을 넘어 임의의 호출 데이터를 다른 체인에 전달하는 일반화된 인터체인 메시징. LayerZero, Axelar, Wormhole, Chainlink CCIP가 대표적. 

// LayerZero 스타일 OApp 예시
contract MyOApp is OAppSender, OAppReceiver {
    function sendMessage(uint32 dstEid, string calldata message)
        external payable
    {
        bytes memory payload = abi.encode(message);
        _lzSend(dstEid, payload, "", MessagingFee(msg.value, 0), payable(msg.sender));
    }

4. 보안 — "브릿지는 가장 약한 고리"

주요 해킹 사례:
  • Ronin Bridge (2022.03): 9개 검증자 중 5개 키 탈취 → 약 6.25억 달러
  • Wormhole (2022.02): Solana guardian 서명 검증 우회 → 약 3.25억 달러
  • Nomad (2022.08): Merkle root 초기화 버그 → 약 1.9억 달러
  • Poly Network (2021.08): keeper 권한 함수 오용 → 약 6.1억 달러
  • Multichain (2023.07): CEO 키 단일 보관 → 약 1.3억 달러
공통 원인: 다자간 신뢰 모델의 키 관리 실패와 크로스체인 검증 로직의 미묘한 버그.

5. 차세대 — zkBridge

소스 체인의 라이트클라이언트 검증을 ZK 회로로 압축해 대상 체인에서 단일 증명으로 검증. Polyhedra, Succinct Labs(SP1), Electron Labs 등이 구현 중. 신뢰가정 없는 보안과 온체인 검증 비용의 트레이드오프를 ZK 증명 압축으로 해결하려는 시도.
