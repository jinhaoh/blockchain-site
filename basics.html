1. 블록체인이란?

블록체인(Blockchain)은 암호학적으로 연결된 블록의 순차적 자료구조를 분산 네트워크의 다수 노드가 공유하고, 합의 알고리즘으로 단일 상태에 도달하게 만드는 분산원장(Distributed Ledger) 기술이다. 중앙 신뢰 기관 없이도 변조 저항성, 이중지불 방지, 최종성을 제공한다.

📚 더 읽기: Bitcoin 백서 (Nakamoto, 2008) · Ethereum 백서 · 위키백과: 블록체인

핵심 명제: 블록체인은 단일 기술이 아니라 해시 함수 + 디지털 서명 + P2P 네트워크 + 합의 알고리즘의 공학적 조합이다.
그림 1. 블록체인 스택 5계층 — 하위 계층은 상위 계층에 신뢰의 뿌리를 제공한다.

🌱 쉽게 이해하기 (비유로 보는 블록체인)

기술 용어가 부담스러우신가요? 4가지 직관적인 그림으로 핵심 개념을 정리했습니다.

비유 1. "모두가 같은 장부를 가지고 있다"

은행은 자기 장부에만 거래를 기록합니다. 블록체인은 네트워크의 모든 사람이 똑같은 장부 복사본을 가지고 있고, 새 거래가 생길 때마다 모두가 동시에 자기 장부에 적습니다.

중앙화: 은행만 장부를 가짐 · 분산화: 모두가 똑같은 장부의 복사본을 가짐

비유 2. "거래 묶음을 도장으로 이어 붙인다"

거래들을 한 묶음(블록)으로 만들고, 그 블록에 도장(해시)을 찍습니다. 다음 블록은 이전 블록의 도장을 자기 안에 포함해서 만들어요. 그래서 사슬처럼 이어집니다.

블록 사슬(chain) — 도장(해시)이 다음 블록과 연결되어 끊을 수 없게 됨

비유 3. "하나만 바꿔도 뒤가 다 깨진다"

누가 과거 거래 하나를 위조하려면, 그 블록의 도장이 바뀝니다. 그러면 그 뒤의 모든 블록을 다시 만들어야 하고, 모든 참여자가 가지고 있는 사본과도 달라져서 즉시 발각됩니다.

변조 저항성 — 하나 바꾸려면 뒤의 모든 블록을 다시 만들고, 모든 참여자의 사본도 바꿔야 함 (사실상 불가능)

비유 4. "다수가 동의한 것이 진실"

여러 명이 동시에 같은 거래를 "맞아"라고 동의해야 새 블록이 추가됩니다. 절반 이상의 참여자가 인정한 거래만 영구히 기록됩니다.

합의(Consensus) — 절반 이상이 인정하면 진실이 됨 · 1명이 거짓말해도 다수에 묻힘
한 줄 요약: 블록체인 = (1) 모두가 같은 장부 보유 + (2) 도장으로 이어진 사슬 + (3) 다수 동의로만 추가. 이 세 가지가 결합되어 "중앙기관 없이도 거래를 신뢰할 수 있는" 시스템이 됩니다.

2. 블록 구조

각 블록은 헤더(Header)와 바디(Body)로 구성된다. 이전 블록의 해시를 헤더에 포함시켜 체인을 형성한다.

📚 참고: Bitcoin Developer Reference · Ethereum.org: Blocks

merkleRoot: a1b2c3...
nonce: 2083236893
Block #1

merkleRoot: d8f3a7...
nonce: 18923
Block #2
prev: 9c1185...

비트코인 블록 헤더 (80 bytes)

struct BlockHeader {
    int32_t  version;          // 4  bytes
    uint8_t  prevBlockHash[32]; // 32 bytes — 이전 블록의 SHA256d
    uint8_t  merkleRoot[32];    // 32 bytes — 트랜잭션 머클 루트
    uint32_t timestamp;        // 4  bytes — Unix epoch
    uint32_t bits;             // 4  bytes — 난이도 타깃 (compact)
    uint32_t nonce;            // 4  bytes — PoW 카운터
};

헤더의 어떤 비트라도 변경되면 SHA256(SHA256(header)) 결과가 완전히 달라지므로 이후 모든 블록의 prevBlockHash가 깨진다. 이것이 변조 저항성의 기술적 근거다.

3. 암호학적 해시 함수

SHA-256, Keccak-256(이더리움) 같은 해시 함수는 다음 세 가지 성질을 만족해야 한다:

📚 참고: NIST FIPS 180-4 (SHA-2) · Keccak Team (SHA-3) · MDN: SubtleCrypto.digest()

  • 전상저항성(Pre-image): h = H(x)에서 x를 역산하기 어려움
  • 2차 전상저항성: H(x) = H(x')인 x' ≠ x 찾기 어려움
  • 충돌저항성: H(x) = H(y)인 임의의 (x, y) 쌍 찾기 어려움
SHA-256 인터랙티브 데모
입력 1글자만 바꿔도 해시 전체가 완전히 달라집니다 (Avalanche Effect)

4. 머클 트리 (Merkle Tree)

블록에 포함된 수천 개의 트랜잭션을 단 32바이트의 머클 루트로 요약하는 이진 해시 트리. SPV 클라이언트는 전체 블록을 다운로드하지 않고도 O(log n) 크기의 머클 증명으로 트랜잭션 포함 여부를 검증할 수 있다.

📚 참고: Wikipedia: Merkle Tree · Bitcoin 백서 §8 (SPV) · Ethereum Patricia Trie

Root = H(H12 ‖ H34)
H12 = H(H1 ‖ H2)
H34 = H(H3 ‖ H4)
H1 = H(Tx1)
H2 = H(Tx2)
H3 = H(Tx3)
H4 = H(Tx4)
// Merkle Proof 검증 (의사 코드)
function verifyProof(txHash, proof, root) {
  let computed = txHash;
  for (const { hash, position } of proof) {
    computed = position === 'left'
      ? sha256(hash + computed)
      : sha256(computed + hash);
  }
  return computed === root;
}

5. 합의 알고리즘

비잔틴 장군 문제(Byzantine Generals Problem)를 분산 환경에서 해결하기 위한 메커니즘.

📚 참고: Ethereum: PoW · Ethereum: PoS · Castro & Liskov: PBFT · Tendermint BFT

그림 2. PoW vs PoS — 보안의 원천이 "에너지"에서 "자본"으로 이동
알고리즘대표 체인최종성비잔틴 내성특징
PoWBitcoin, Litecoin확률적≤ 50%높은 보안, 에너지 큼

// 채굴은 다음 조건을 만족하는 nonce를 찾는 작업
while (sha256d(blockHeader) > target) {

주의: 51% 공격은 이론적 임계점일 뿐, 실제로는 25~33%의 해시파워만으로도 셀피시 마이닝(Eyal & Sirer, 2014)이 가능하다.

PoW vs PoS — 흔한 오해와 균형 잡힌 시각

PoW는 단순히 "환경에 나쁜 구식 기술"이 아니다. 양자의 트레이드오프를 정확히 이해해야 한다.

PoW의 장점 (자주 간과됨)

  • 물리적 보안 비용: 공격하려면 실제 에너지·하드웨어 투입이 필요 → 비용 명확
  • 잉여·재생에너지 흡수: 일부 채굴은 송전 불가 지역 잉여 전력·천연가스 플레어를 활용 (Cambridge CCAF 2022 추정 PoW 채굴의 50%+가 재생/저탄소 에너지)
  • "무관여 발행" (Permissionless 발행): 누구나 채굴 진입 가능, 토큰 사전 분배 불필요
  • 오랜 검증 기간: 15년 이상 메인넷 운영으로 보안 모델 입증

PoS의 단점 (자주 간과됨)

  • 부의 집중 → 권력 집중: "많이 가진 자가 더 많이 결정" — 토큰 분배의 공정성이 핵심 변수
  • 스테이킹 풀 과점: Lido·Coinbase·Binance 등 상위 5개 스테이커가 Ethereum 검증의 50%+ 차지
  • 슬래싱·운영 복잡성: 검증자 운영 실수로 자산 손실 위험
  • "무관여 발행" 불가: 초기 토큰 분배가 정의되어야 시작 가능
  • Long-Range Attack: 과거 키 누출 시 대안 체인 생성 가능 (Weak Subjectivity 필요)

결론

"PoS가 우월" 또는 "PoW가 우월"이라는 단순 진술은 부정확하다. 다른 보안 가정·다른 거버넌스 트레이드오프를 가진 두 모델로 보는 것이 정확하다. Bitcoin이 PoW를 유지하고 Ethereum이 PoS로 전환한 것은 각자의 가치 우선순위(보안 정통성 vs 환경·확장성)가 다르기 때문이다.

6. 블록체인의 종류와 분류

블록체인은 단일 기술이 아니라 다양한 변형이 존재한다. 가장 중요한 분류 축은 (1) 접근 권한, (2) 계층 구조(Layer), (3) 목적·특화 영역이다. 유스케이스에 따라 적합한 종류가 다르다.

6.1 접근 권한별 (Permissionless vs Permissioned)

누가 노드를 운영하고 트랜잭션을 검증할 수 있는가에 따른 분류. 가장 본질적인 구분이다.

접근 권한 스펙트럼 — 왼쪽일수록 개방적, 오른쪽일수록 통제적
특성PermissionlessConsortiumPermissioned
노드 운영누구나승인된 기관 다수단일·소수 운영자
합의 알고리즘PoW, PoSBFT (Tendermint, IBFT)Raft, BFT, 또는 신뢰 기반
처리량 (TPS)낮음 (수십~수천)중간 (수천)높음 (수만+)
토큰 인센티브필수 (보안의 원천)선택적대개 없음
적용 영역암호화폐, DeFi, NFT국제송금, CBDC기업 내부, 공급망

6.2 계층별 (Layer Architecture)

최근 블록체인 생태계는 단일 체인이 모든 것을 하지 않고, 역할을 계층으로 나누어 분산한다.

블록체인 계층 구조 — 하위는 보안·범용성, 상위는 특화·고성능 제공

각 계층의 역할

계층역할대표 예시비고
L0 체인 간 상호운용성, 공유 보안 제공 Polkadot, Cosmos Hub, Avalanche "체인의 체인"
L1 독자 합의로 보안·최종성 제공 Bitcoin, Ethereum, Solana 가장 신뢰의 뿌리
L2 L1 위에서 TPS·수수료 개선 Arbitrum, Optimism, zkSync, Lightning L1에 보안 의존
L3 L2 위에 특정 앱·기업용 환경 Arbitrum Orbit, Caldera, Conduit 주로 게임·엔터프라이즈

L2의 세부 분류

유형방식대표 예시

Sidechain 독립 합의를 가진 별도 체인 (L1에 보안 의존 X) Polygon PoS, Gnosis Chain
Validium ZK 증명 + 오프체인 데이터 가용성 Immutable X, dYdX v3
Plasma (역사적) 초기 L2 시도, 데이터 가용성 문제로 쇠퇴 OMG Network (구)
